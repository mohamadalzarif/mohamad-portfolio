# Reference Repositories

These sources are for inspiration and implementation judgment only. Do not copy code blindly, install libraries from them by default, or expand dependencies without a concrete need.

## Next.js App Router Structure

- [vercel/next.js examples](https://github.com/vercel/next.js/tree/canary/examples): canonical examples for App Router conventions, metadata, routing, and deployment expectations.
- [shadcn-ui/ui](https://github.com/shadcn-ui/ui): reference for component composition, accessible primitives, and Tailwind-based UI patterns.

## Editorial and Documentation Feel

- [withastro/docs](https://github.com/withastro/docs): strong documentation information architecture, sidebar hierarchy, and readable technical writing.
- [vercel/geist-font](https://github.com/vercel/geist-font): useful reference for restrained typography and modern product/editorial presentation.

## Accessibility and Interaction Standards

- [radix-ui/primitives](https://github.com/radix-ui/primitives): source reference for accessible component behavior, keyboard interaction, and ARIA patterns.
- [framer/motion](https://github.com/framer/motion): reference for restrained Motion for React usage already allowed by the project stack.

## Portfolio and Case Study Inspiration

- [leerob/leerob.io](https://github.com/leerob/leerob.io): useful for clean Next.js portfolio organization and concise content structure.
- [bchiang7/v4](https://github.com/bchiang7/v4): useful as a portfolio reference, mainly for project organization and content discipline rather than visual style.

## Usage Notes

- Treat these as references for structure, accessibility, and editorial clarity.
- Keep the “Fieldnotes & Systems” direction distinct from generic developer portfolio templates.
- Add dependencies only when a required behavior cannot be built well with the existing stack.
